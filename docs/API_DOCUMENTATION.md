# UjenziXform API Documentation

## Overview

UjenziXform uses **Supabase** as its backend, providing:
- PostgreSQL database with Row Level Security (RLS)
- Real-time subscriptions
- Authentication (email/password, OAuth)
- Edge Functions for serverless logic
- Storage for files and images

**Base URL:** `https://wuuyjjpgzgeimiptuuws.supabase.co`

## Authentication

### Sign Up
```typescript
const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'securepassword123',
  options: {
    data: {
      full_name: 'John Doe',
      role: 'builder' // 'builder' | 'supplier' | 'delivery'
    }
  }
});
```

### Sign In
```typescript
const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'securepassword123'
});
```

### Sign Out
```typescript
const { error } = await supabase.auth.signOut();
```

### Get Current User
```typescript
const { data: { user } } = await supabase.auth.getUser();
```

### Get Session
```typescript
const { data: { session } } = await supabase.auth.getSession();
```

## Database Tables

### profiles
User profile information.

| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| user_id | uuid | References auth.users |
| email | text | User email |
| full_name | text | Full name |
| phone | text | Phone number |
| role | text | 'builder', 'supplier', 'delivery', 'admin' |
| county | text | Kenya county |
| profile_photo_url | text | Profile image URL |
| is_professional | boolean | Professional builder flag |
| user_type | text | 'individual', 'company' |
| created_at | timestamp | Creation date |

**Example Query:**
```typescript
const { data, error } = await supabase
  .from('profiles')
  .select('*')
  .eq('user_id', userId)
  .single();
```

| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| user_id | uuid | References profiles |
| business_name | text | Company name |
| business_type | text | Type of business |
| registration_number | text | Business registration |
| kra_pin | text | KRA PIN number |
| county | text | Operating county |
| physical_address | text | Business address |
| categories | text[] | Material categories |
| delivery_available | boolean | Offers delivery |
| minimum_order | numeric | Minimum order value |
| rating | numeric | Average rating |
| verified | boolean | Verification status |
| created_at | timestamp | Registration date |

**Example Query:**
```typescript
// Get all verified suppliers in Nairobi
const { data, error } = await supabase
  .from('suppliers')
  .select('*, profiles(*)')
  .eq('county', 'Nairobi')
  .eq('verified', true)
  .order('rating', { ascending: false });
```

| status | text | 'pending', 'confirmed', 'shipped', 'delivered' |
| qr_code_url | text | QR code for tracking |

```typescript

  .from('purchase_orders')
  .insert({
    po_number: `PO-${Date.now()}`,
```

## Real-time Subscriptions

### Subscribe to Delivery Updates
```typescript
const subscription = supabase
  .channel('delivery-updates')
  .on(
    'postgres_changes',
    {
      event: '*',
      schema: 'public',
      table: 'delivery_requests',
      filter: `builder_id=eq.${userId}`
    },
    (payload) => {
      console.log('Delivery update:', payload);
    }
  )
  .subscribe();

// Cleanup
subscription.unsubscribe();
```

### Subscribe to New Orders
```typescript
const subscription = supabase
  .channel('new-orders')
  .on(
    'postgres_changes',
    {
      event: 'INSERT',
      schema: 'public',
      table: 'purchase_orders',
      filter: `supplier_id=eq.${supplierId}`
    },
    (payload) => {
      console.log('New order:', payload.new);
    }
  )
  .subscribe();
```

## Row Level Security (RLS) Policies

All tables have RLS enabled. Key policies:

### profiles
- Users can read their own profile
- Users can update their own profile
- Admins can read all profiles

### suppliers
- Public can read verified suppliers
- Suppliers can update their own data
- Admins have full access

### purchase_orders
- Buyers can read/create their orders
- Suppliers can read orders sent to them
- Admins have full access

### delivery_requests
- Builders can manage their requests
- Drivers can read assigned deliveries
- Admins have full access

## Error Handling

All API calls return standardized errors:

```typescript
interface SupabaseError {
  message: string;
  code: string;
  details: string | null;
  hint: string | null;
}

// Example error handling
const { data, error } = await supabase
  .from('suppliers')
  .select('*');

if (error) {
  console.error('Error code:', error.code);
  console.error('Message:', error.message);
  
  // Common error codes
  // PGRST116 - No rows returned
  // 23505 - Unique constraint violation
  // 42501 - Permission denied (RLS)
}
```

## Best Practices

1. **Always use RLS** - Never bypass security policies
2. **Validate input** - Use Zod schemas for validation
3. **Handle errors gracefully** - Show user-friendly messages
4. **Use TypeScript** - Leverage generated types from Supabase
5. **Optimize queries** - Select only needed columns
6. **Cache when possible** - Use React Query for caching
