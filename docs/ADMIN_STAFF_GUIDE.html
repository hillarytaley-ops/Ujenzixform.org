📋 Creating New Admin Credentials

1 Open the Hash Generator Tool

Location: scripts/generate-admin-hash.html

Navigate to your project folder, find the file, and double-click to open in your browser.

2 Enter New Admin Details

Field Example Required
Admin Email john.kamau@company.com ✅ Yes
Full Name John Kamau ✅ Yes
Staff Code UJPRO-2025-JOHN01 ✅ Yes
Department Operations ❌ Optional

3 Generate Hash & SQL

Click the "Generate Hash & SQL" button. The tool will display the encrypted hash and a ready-to-use SQL statement.

4 Run SQL in Supabase

  1. Open Supabase Dashboard
  2. Select your project
  3. Click SQL Editor
  4. Paste the generated SQL
  5. Click Run

5 Verify the New Admin

SELECT email, full_name, is_active,
       CASE WHEN staff_code_hash IS NOT NULL 
            THEN '✅ Ready' ELSE '❌ Missing' END as status
FROM admin_staff
ORDER BY created_at DESC;

🔄 Resetting a Staff Code

If an admin forgets their code or it's compromised:

  1. Open scripts/generate-admin-hash.html
  2. Create a new staff code for the same email
  3. Run this SQL in Supabase:
UPDATE admin_staff
SET staff_code_hash = 'new-64-character-hash-here',
    updated_at = NOW()
WHERE email = 'john.kamau@company.com';

🚫 Deactivating an Admin

Temporarily disable (keeps record):

UPDATE admin_staff
SET is_active = false
WHERE email = 'john.kamau@company.com';

Reactivate:

UPDATE admin_staff
SET is_active = true
WHERE email = 'john.kamau@company.com';

🗑️ Permanently Removing an Admin

⚠️ Warning: This cannot be undone! 
DELETE FROM admin_staff
WHERE email = 'john.kamau@company.com';

🔒 Security Best Practices

✅ DO

  • Use unique staff codes for each admin
  • Share codes via secure channels only
  • Deactivate accounts when staff leave
  • Rotate codes every 6-12 months
  • Keep the hash generator tool secure

❌ DON'T

  • Share staff codes via email
  • Use simple codes like "admin123"
  • Share same code with multiple admins
  • Store plain-text codes anywhere
  • Share the SHA-256 hash

🆘 Troubleshooting

"Access Denied" Error

  1. Check if email is spelled correctly
  2. Verify staff code (case-sensitive)
  3. Check if account is active in database

"Account Locked" Error

  • Wait 10 minutes for automatic unlock
  • Or clear in browser console (F12 → Console):
localStorage.removeItem("admin_lockout_until");
localStorage.removeItem("admin_login_attempts");
