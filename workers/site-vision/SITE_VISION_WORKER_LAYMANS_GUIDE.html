Site vision worker — plain-language guide

UjenziXform / MradiPro · workers/site-vision · For printing: use your browser Print (Ctrl+P) and choose Save as PDF.

What this thing is

You have a small program that runs on a laptop or mini PC at the site (or office). It:

  1. Looks up your cameras in the database (Supabase).
  2. For each camera, tries to grab a snapshot from the video stream (not the website link).
  3. Notices movement (and optionally uses AI for people/cars).
  4. Writes short notes into the database (“something moved on camera X at this time”).

Your website (Analytics) reads those notes so you see charts and lists. The website does not process the video itself.

Python version

The program is written in Python. On Windows, Python 3.11 or 3.12 is the least fussy; 3.13 can work if the install steps in the README were followed.

Before you start (prerequisites)

  1. Database table — Someone added a table for “vision events” in Supabase (migration). Without it, the program has nowhere to save notes.
  2. That table must exist in your Supabase project.
  3. Camera links must be real video addresses — Like a direct TV channel address (RTSP, or a special web link ending in .m3u8), not a normal “watch in browser” page like GoPro’s share link.

GoPro / gopro.com links

A share page is like a YouTube watch page: it’s for people clicking in a browser. The worker needs something like the raw feed the player uses behind the scenes — and GoPro’s public share link usually isn’t that simple address.

So: you can’t paste the same link you email to friends and expect this script to work.

Step-by-step (simple version)

1. Test in VLC — VLC is a free video player. If you paste a URL and you see live camera video, that URL is probably OK. If you get a webpage, login, or error, it’s not a direct stream.

2. IP cameras — Many security cameras give an RTSP address — a long rtsp://... line with IP, user, password. It’s in the manual or camera app. The README shows examples; your camera might use a slightly different path.

3. HLS — Some recorders give a link ending in .m3u8. Same rule: if VLC plays it, you’re good.

4. GoPro limits — The cloud gallery link isn’t a stream for this tool. Real options: use other cameras with RTSP, or a more complex live-stream setup. For most teams, IP cameras on site are the practical fix.

5. Save in Supabase — Open your project → Table Editor → cameras → each camera row → put the working VLC URL in stream_url.

6. Restart the program — Stop it (Ctrl+C), start it again so it picks up new URLs.

Setup & run (what the commands mean)

  • venv — A private toolbox for this project so Python packages don’t clash with other projects.
  • pip install -r requirements.txt — Install the listed libraries (OpenCV, Supabase client, etc.).
  • .env — A secret notepad with your Supabase address and service key (server-only key, never in the website).
  • python main.py — Start the worker; leave the window open while it should keep running.

Environment variables (the knobs)

  • How often it checks all cameras again (seconds).
  • How sensitive “motion” is (higher = fewer alerts).
  • How long to wait before logging the same kind of event again (so the database isn’t spammed).
  • YOLO — Optional smarter detection (people, vehicles); off by default.

Security

The service role key is like a master key to your database. Only this worker (or trusted servers) should have it — never in the public website or Vercel’s “browser” settings.

Operations (many cameras, network)

  • Many cameras — One round of “check each camera once” per cycle is normal; if the PC struggles, slow down (longer interval between rounds).
  • RTSP on LAN — The PC running the script should be on the same Wi-Fi/network as the cameras if you use local 192.168... addresses.
  • HLS in the cloud — If the URL is on the public internet, the worker can run anywhere that can reach that URL.

Troubleshooting (short)

  • proxy error — Wrong mix of library versions; reinstall from requirements.txt as the README says.
  • pyiceberg build error — Don’t use a too-new Supabase package on Windows without dev tools; use the pinned versions.
  • Cannot open stream — Wrong URL type or network; VLC test + fix stream_url.
  • No data in the app — You might be logged in as admin in the browser only without a real Supabase login; database rules may block reads.
One-sentence summary: A small Python job on a PC pulls real video stream URLs from your database, watches for motion, saves short events to Supabase, and your Analytics page shows them — but the URLs must be real streams (test in VLC), not GoPro gallery web pages.
